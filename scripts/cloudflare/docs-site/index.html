RAG System API Documentation

Cloudflare-hosted RAG system with govinfo and congress data integration

Overview

This RAG (Retrieval-Augmented Generation) system provides APIs for searching and managing documents from government sources including govinfo.gov and congress.gov. The system is hosted on Cloudflare Workers and integrates with the autorag database.

Base URL

https://api.rag-cloudflare.pages.dev

Authentication

Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

API Endpoints

GET /api/health

Check system health and connectivity

Response:

{
  "status": "healthy",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "services": {
    "kv": "connected"
  }
}

POST /api/search

Search documents using natural language queries

Request Body:

{
  "query": "climate change legislation",
  "limit": 10,
  "filters": {
    "source": "govinfo",
    "date_from": "2024-01-01"
  }
}

Response:

{
  "success": true,
  "query": "climate change legislation",
  "results": [...],
  "total": 5
}

GET /api/documents

Retrieve documents with pagination

Query Parameters:

  • limit (optional): Number of documents to return (default: 10)
  • offset (optional): Number of documents to skip (default: 0)

Response:

{
  "success": true,
  "documents": [...],
  "limit": 10,
  "offset": 0,
  "total": 25
}

POST /api/ingest

Ingest new documents into the system

Request Body:

{
  "documents": [
    {
      "title": "Document Title",
      "content": "Document content...",
      "summary": "Brief summary",
      "metadata": {
        "source": "govinfo",
        "url": "https://example.com"
      }
    }
  ],
  "source": "api"
}

Response:

{
  "success": true,
  "message": "1 documents ingested successfully",
  "stored_count": 1
}

Data Sources

  • govinfo.gov: Official government publications and documents
  • congress.gov: Congressional bills, resolutions, and records
  • opendiscourse.net: Parliamentary proceedings and debates

Integration with AutoRAG

This system integrates with the opendiscourse.net-cf AutoRAG database to provide enhanced document retrieval and question-answering capabilities. Documents ingested through this API are automatically indexed and made available to the AutoRAG system.

Error Handling

All API endpoints return standardized error responses:

{
  "error": "Error Type",
  "message": "Detailed error message"
}

Rate Limits

API requests are subject to Cloudflare Workers rate limits. For high-volume usage, consider implementing caching or batch processing.
